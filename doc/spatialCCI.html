Simulating Spatial Cell-Cell Interactions

scMultiSim can simulate spatial cell-cell interactions. To do so, we need to provide the cci option as a list. The following code will print more instructions on how to use the cci option.

library(scMultiSim)

scmultisim_help("cci")

## 
## To enable simulating cell-cell interaction, the value should be a list including
## the following names:
## 
## - params: (data.frame)
##     The spatial effect between neighbor cells.
##     It should be a data frame similar to the GRN parameter.
## - step.size: (number, optional)
##     If using continuous population, use this step size to further divide the
##     cell types on the tree. For example, if the tree only has one branch 1 -> 2
##     and the branch length is 1 while the step size is 0.34, there will be totally
##     three cell types: 1_2_1, 1_2_2, 1_2_3.
## - cell.type.interaction: ("random" or a matrix)
##     The interaction level between different cell types.
##     They act as factors multiplied to the ligand effect.
##     Supply the string "random" to let scMultiSim generate these factors randomly.
##     Otherwise, use cci_cell_type_params() to generate the template data structure.
##     See the help of this method for more info.
## - max.neighbors: (integer from 1 to 4, optional)
##     Constraint the maxinum number of neighbors with CCI for each cell.
##     The neighbors with CCI will be randomly sampled. 
## 

## NULL

Now, we prepare a ligand-receptor interaction database. This is pretty similar to the GRN network: it is a data frame with three columns, specifying target, regulator, and effect, respectively. The target and regulator columns should contain the IDs of the target and regulator genes. In the following example, we have two ligand-receptor pairs interacting between two neighboring cells.

lig_params <- data.frame(
  target    = c(101, 102),
  regulator = c(103, 104),
  effect    = c(5.2, 5.9)
)

We can now simulate the spatial cell-cell interactions. In scMultiSim, the CCI network is cell-type based, which means that between each cell type pair, we can have a different CCI network sampled from the database defined above.

data(GRN_params_100)

options_ <- list(
  rand.seed = 0,
  GRN = GRN_params_100,
  num.genes = 200,
  num.cells = 300,
  num.cifs = 50,
  tree = Phyla3(),
  discrete.cif = T,
  discrete.pop.size = c(70, 100, 130) %>% as.integer(),
  cci = list(
    params = lig_params,
    cell.type.interaction = "random"
  )
)

results <- sim_true_counts(options_)

## CCI simulation is enabled.
## Get CIF...Get params...Done
## Simulating...50..100..150..200..250..300..
## Time spent: 9.94 mins

In the above example, we set discrete.cif to TRUE, so the cell population is divided into three discrete cell types, with 70, 100, and 130 cells, respectively. We set cell.type.interaction to random, so the CCI network between each cell type pair is randomly sampled from the database. How many LR pairs are sampled for each cell type pair is controlled by the cell.type.lr.pairs option. For example, if you have 8 LR pairs in the database, and set cell.type.lr.pairs to 3:5, then for each cell type pair, three to five LR pairs will be randomly sampled from the database.

You may also directly specify the CCI network for each cell type pair. See the next subsection for an example.

The results$cell_meta will contain the cell type information used in CCI. We can plot the cell spatial locations using plotCellLoc(). The arrows indicate cell-cell interactions between two cells (for the first ligand-receptor pair).

plot_cell_loc(results)

The cell locations are available in results$cci_locs.

Continuous Cell Population

The above example shows how to simulate spatial cell-cell interactions between discrete cell types. We can also simulate CCI for continuous trajectories. However, since the CCI ground truth is cell-type based, we need discretize the differentiation tree to get the cell types. Here, we use another parameter step.size to specify the length of each segment in the differentiation tree. For example, if you use a linear tree of length 2 (Phyla1(len = 2)) and set the step.size to 0.5, then there will be 4 cell types (the tree is divided into segments of length 0.5), each segment is treated as a cell type in CCI.

options_ <- list(
  rand.seed = 0,
  GRN = GRN_params_100,
  num.genes = 200,
  num.cells = 300,
  num.cifs = 50,
  tree = Phyla3(),
  cci = list(
    params = lig_params,
    max.neighbors = 4,
    cell.type.interaction = "random",
    step.size = 0.5
  )
)

In the results cell_meta data frame, the column cell.type and cell.type.idx will contain the cell type information.

Specifying CCI Network for Each Cell Type Pair

In the above example, we set cell.type.interaction to random, so the CCI network between each cell type pair is randomly sampled from the database. If you want to specify the CCI network for each cell type pair, you can set cell.type.interaction to an array of dimension n_cell_types x n_cell_types x n_lr, where n_cell_types is the number of cell types, and n_lr is the number of ligand-receptor pairs in the database. If you are not sure about the dimension of the array, you can generate an empty matrix by running cci_cell_type_params():

ctp <- cci_cell_type_params(
        Phyla3(), # the same tree used when simulating
        6, # num_lr: number of ligand-receptor pairs in the database
        # (optional) step.size = 0.5: the same parameter used when simulating; to calculate the correct number of cell types
        # (optional) discrete = FALSE: the same parameter used when simulating; to calculate the correct number of cell types
)

dim(ctp$params)

## [1] 5 5 6

Then ctp will be an array of dimension n_cell_types x n_cell_types x n_lr, where ctp[i, j, k] == 1 represents the k-th ligand-receptor pair is interacting between cell type i and j. You can then fill this matrix to specify the CCI network you want to use, e.g., ctp$params[1, 2, 1] <- 1, then in your options use cell.type.interaction = ctp.

Additional Spatial Layouts

In addition to the default spatial layout, we also provide several other commonly-used spatial layouts, for example layers and islands.

When using islands, you need to specify which cell types form the islands: layout = "islands:1,2" means cell type 1 and 2 are the island cell types.

options_ <- list(
  GRN = NA,
  num.genes = 200,
  num.cells = 300,
  tree = Phyla3(),
  discrete.cif = T,
  discrete.pop.size = c(70, 100, 130) %>% as.integer(),
  cci = list(
    params = lig_params,
    cell.type.interaction = "random",
    grid.size = 25,
    layout = "layers"
  )
)

results <- sim_true_counts(options_)

## CCI simulation is enabled.
## Get CIF...Get params...Done
## Simulating...50..100..150..200..250..300..
## Time spent: 9.86 mins

plot_cell_loc(results)


tree4 <- ape::read.tree(text = "(A:0.1, B:0.1, (C:0.1, D:0.1):0.1);")

options_ <- list(
  GRN = NA,
  num.genes = 200,
  num.cells = 500,
  tree = tree4,
  discrete.cif = T,
  discrete.pop.size = c(50, 60, 200, 190) %>% as.integer(),
  cci = list(
    params = lig_params,
    cell.type.interaction = "random",
    grid.size = 25,
    layout = "islands:1,2"
  )
)

results <- sim_true_counts(options_)

## CCI simulation is enabled.
## Get CIF...Get params...Done
## Simulating...50..100..150..200..250..300..350..400..450..500..
## Time spent: 26.72 mins

plot_cell_loc(results, show.arrows = FALSE)

Specify the Cell-Cell Communication Range

By default, we only consider the cell-cell interactions between a cell and its (at most) four neighboring cells. You can change this behavior to model the cell-cell interactions between more cells within a certain range. Simply set the radius option in cci to the desired value. (Note that currently only the layers and islands layouts supports this option.)

Additionally, since now a cells neighborhood can be very large, you can specify the maxinum number of neighbors used in the simulation by setting the max.neighbors option in cci. Default is 4.

options_ <- list(
  GRN = NA,
  num.genes = 200,
  num.cells = 1000,
  tree = Phyla3(),
  discrete.cif = T,
  discrete.pop.size = c(70, 100, 130) %>% as.integer(),
  cci = list(
    params = lig_params,
    cell.type.interaction = "random",
    grid.size = 40,
    radius = 4,  # cells within distance 4 are considered neighbors
    max.neighbors = 8,  # 8 neighbors in the neighborhood will be sampled for each cell
    layout = "layers"
  )
)

Obtaining Single-cell CCI ground Truth

In the above examples, we only provide the cell-type-level CCI ground truth, while the interaction between single-cells are sampled from the cell-type-level ground truth. It is possible to fix the single-cell-level CCI ground truth by setting the single.cell.gt option to TRUE.

If enabled, the result object will have a new field cci_gt, a n_cells x n_cells x n_lr array, where cci_gt[i, j, k] == 1 means the k-th ligand-receptor pair is interacting between cell i and j.
